Technical Overview · High-Level

How the platform actually works.

A five-stage evaluation pipeline behind a single secure edge, with real agent telemetry, calibrated scoring, and a closed remediation loop — built to run from a laptop to full production without changing a line of code.

01 — What Verity Offers

A single control plane for agent assurance.

One public API edge fronts a pipeline of specialised services. Light work resolves instantly; heavy work runs asynchronously on a durable queue with live progress streamed back.

EdgeOne public API + WebSocket
The only entry point. It authenticates the tenant, freezes an immutable job, and streams progress — no business logic, so it can’t drift.
PipelineFive capability services
Persona → Question → Run → Verdict → Evidence, each isolated and independently scalable.
TelemetryReal OpenTelemetry spans
The agent-under-test emits an industry-standard OpenInference span tree per turn — the same trace format your APM already speaks.
Verdict layerSix trust pillars + deploy gate
Calibrated LLM juries score every answer, roll up into six pillars, and produce one go / no-go gate plus an audit-ready evidence pack.

04 — Technical Benefits

Engineered to be trusted.

Laptop → production
Runs zero-infra in one process locally; flip one switch to point identical code at ScyllaDB, NATS & MinIO.
Provider-agnostic
A model fallback chain (GPT-4 → open-weight → offline) means no vendor lock-in, and it always boots.
Immutable & versioned
Personas, runs, verdicts & evidence are append-only — any historical decision reconstructs exactly.
Idempotent by design
Identical inputs return the original result; redelivery is a no-op — safe to retry, cheap to re-run.
Tenant-isolated
Tenancy is physical — the top-level partition on every key — resolved once at the edge.
Standards-native
Real OpenTelemetry / OpenInference traces plug into the observability stack you already run.

Synthetic data only — no real customer data anywhere, including demos.
