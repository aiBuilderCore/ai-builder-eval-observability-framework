Create persona — cast a character

A persona is a small, deliberate fiction. The more specifically it misbehaves, the more useful it is for grading.

01Identity

Give the persona a memorable, alliterative name — it makes them easier to talk about across runs.

02Demographics

Synthetic, and clearly so — fake names, broad bands. Never reuse anything that resembles a real customer profile.

03Tone

How does this persona sound? Tone shapes the question-generation strategy and how strict the judges should be on register.

04Goals

What is the persona trying to accomplish? Plain language. One bullet per goal.

05Edge-case behaviors

The reason this persona earns its keep. The more specifically they misbehave, the more useful they are.

06Tags & version

Tags drive filtering and rubric routing. Versioning is required — bumping it on a meaningful change keeps eval runs reproducible.
