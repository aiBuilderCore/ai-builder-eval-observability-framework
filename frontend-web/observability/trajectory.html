Trajectory drift · stage 4 of feedback loop · identify failure modes

Drift on the shape of the trajectory — not the scalars

Token / latency / cost dashboards answer is this slower than yesterday? Trajectory drift answers is the agent picking different tools, planning differently, delegating differently — even when latency and cost look fine. This is the failure mode that's invisible everywhere else in the market.

What this tells you: the frequency of each tool the agent invoked in the baseline batch vs the candidate batch. A big shift here — even with the same latency — means the agent is calling different tools.
