Monitor catalogue — OTLP/gRPC · OTLP/HTTP · same trace contract

Authored monitors watch every live span under the same dual-emit OpenInference + OTel GenAI semconv schema the simulation-first path uses. Four signal kinds: span (latency, cost, token budget), score (judge pass rate, κ drift), trajectory (edit-distance, KL divergence), anomaly (TRAIL taxonomy counts). State machine: armed → firing → open → acknowledged → resolved.

Scope — where does this data come from?

This page shows production traffic — traces arriving live from the agent's real users, over OTLP. They land under the same OpenInference + OTel GenAI span contract the pre-release batches use, so anything you observe here can be reproduced by submitting a batch. If a monitor fires and you want to confirm the fix, run a batch first.
